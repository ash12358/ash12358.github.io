git基本使用
发表于 2019-11-19| 更新于 2019-11-25
阅读量:

理想中的作业流程

  1. 在本机上通过pycharm写代码，然后通过git这个工具把代理同步到web端。
  2. 然后在服务器上使用git pull下来，然后运行。
  3. 如果在服务器端运行的过程中出现了什么问题，再回到本机上修改代码，再push到web端。
  4. 然后再在服务器上pull下来，运行。

大体就是这么个想法。

基本使用

先通过网页，在github上新建一个仓库。
然后可以通过git clone https://xxx.git克隆这个仓库，也可以现在本地做一些工作，然后通过git remote add origin https://xxx.git来关联远程仓库。
然后就是些基本命令：

  1. git add .将当前工作目录中的所有文件添加到工作区
  2. git commit -m "." 提交到（我也不知道是哪）
  3. git push推到web端
  4. git pulldown都服务器上

其他命令

  1. git branch branch_name新建一个分支
  2. git checkout branch_name切换分支
